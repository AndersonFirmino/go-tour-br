    Introdução

    Pacotes

    Cada programa Go é composto de pacotes.

    Programas começam rodando pelo pacote main.

    Este programa está usando os pacotes com caminhos de importação "fmt" e "math".

    Por convenção, o nome do pacote é o mesmo que o último elemento do caminho de importação.

    package main import ( "fmt" "math" ) func main() { fmt.Println("Dia", math.Pi, "Feliz") }

    Importações

    Este grupo de códigos das importações em um parênteses, a declaração de importação "consignada". Você também pode escrever várias declarações de importação, tais como: 

    	import "fmt"
	import "math"
    mas é comum usar a forma consignada para eliminar a desordem.
    package main import ( "fmt" "math" ) func main() { fmt.Printf("Agora você têm %g problemas.", math.Nextafter(2, 3)) }

    Nomes exportados

    Depois de importar um pacote, você pode consultar os nomes que exporta.

    Em Go, um nome é exportado se começa com uma letra maiúscula.

    Foo é um nome exportados, assim como FOO. O nome foo não é exportado.

    Execute o código. Em seguida, renomeie math.pi para math.Pi e tente novamente.

    package main import ( "fmt" "math" ) func main() { fmt.Println(math.pi) }

    Funções

    A função pode ter zero ou mais argumentos.

    Neste exemplo, adicione dois parâmetros do tipo int.

    Observe que o tipo vem após o nome da variável.

    (Para saber mais sobre o porquê dos tipos serem assim, consulte o artigo sobre a sintaxe de declaração de Go.)

    package main import "fmt" func adicione(x int, y int) int { return x + y } func main() { fmt.Println(adicione(42, 13)) }

    Funções

    Quando dois ou mais consecutivos parâmetros da função nomeados compartilhar um tipo, você pode omitir o tipo de todos, menos o último.

    Neste exemplo, vamos encurtar 

    x int, y int

    para 

    x, y int
    package main import "fmt" func adicione(x, y int) int { return x + y } func main() { fmt.Println(adicione(42, 13)) }

    Funções

    Uma função pode retornar qualquer número de resultados.

    Esta função retorna duas strings.

    package main import "fmt" func troque(x, y string) (string, string) { return y, x } func main() { a, b := troque("olá", "mundo") fmt.Println(a, b) }

    Funções

    Funções têm parâmetros. Em Go as funções podem retornar vários "parâmetros de resultado", e não apenas um único valor. Eles podem ser nomeados e atuam como variáveis.

    Se os parâmetros de resultado são nomeados, uma declaração return sem argumentos retorna os valores atuais dos resultados.

    package main import "fmt" func separe(soma int) (x, y int) { x = soma * 4/9 y = soma - x return } func main() { fmt.Println(separe(17)) }

    Variáveis

    A instrução var declara uma lista de variáveis, como em listas de argumentos de função, o tipo é passado.

    package main import "fmt" var x, y, z int var c, python, java bool func main() { fmt.Println(x, y, z, c, python, java) }

    Variáveis

    A declaração var pode incluir inicializadores, uma por variável.

    Se um inicializador está presente, o tipo pode ser omitido; a variável terá o tipo do inicializador.

    package main import "fmt" var x, y, z int = 1, 2, 3 var c, python, java = true, false, "não!" func main() { fmt.Println(x, y, z, c, python, java) }

    Variáveis

    Dentro de uma função, a instrução := de atribuição curta pode ser utilizada em lugar de um var declaração com o tipo implícito.

    (Fora de uma função, cada construção começa com uma palavra-chave e := não está disponível)

    package main import "fmt" func main() { var x, y, z int = 1, 2, 3 c, python, java := true, false, "não!" fmt.Println(x, y, z, c, python, java) }

    Constantes

    Constantes são declaradas como variáveis, mas com a palavra-chave const.

    Constantes podem ser seqüências de caracteres, booleanos, ou valores numéricos.

    package main import "fmt" const Pi = 3.14 func main() { const Mundo = "世界" fmt.Println("Olá", Mundo) fmt.Println("Dia", Pi, "Feliz") const Verdade = true fmt.Println("Go é legal?", Verdade) }

    Constantes Numéricas

    As constantes numéricas são valores de alta precisão.

    Uma constante sem tipo tem o tipo necessário para o seu contexto.

    Tente imprimir precisadeInt(Grande) também.

    package main import "fmt" const ( Grande = 1<<100 Pequeno = Grande>>99 ) func precisadeInt(x int) int { return x*10 + 1 } func precisadeFloat(x float64) float64 { return x*0.1 } func main() { fmt.Println(precisadeInt(Pequeno)) fmt.Println(precisadeFloat(Pequeno)) fmt.Println(precisadeFloat(Grande)) }

    For

    Go tem apenas uma construção de laço, o for.

    O for básico parece com o que o de C ou Java fazem, exceto que as ( ) se foram (não são nem mesmo opcionais) e os { } são obrigatórios.

    package main import "fmt" func main() { soma := 0 for i := 0; i < 10; i++ { soma += i } fmt.Println(soma) }

    For

    Como em C ou Java, você pode deixar as declarações do começo e do final vazias.

    package main import "fmt" func main() { soma := 1 for ; soma < 1000; { soma += soma } fmt.Println(soma) }

    For

    Nesse ponto, você pode tirar as vírgulas: o while do C é escrito com for em Go.

    package main import "fmt" func main() { soma := 1 for soma < 1000 { soma += soma } fmt.Println(soma) }

    For

    Se você omitir a condição do laço, ele executará infinitamente.

    package main func main() { for ; ; { } }

    For

    E sem cláusulas em tudo o ponto-e-vírgula pode ser omitido, e um loop infinito é compactadamente expresso.

    package main func main() { for { } }

    If

    A declaração if parece com o que o de C ou Java fazem, exceto que as ( ) se foram (não são nem mesmo opcionais) e os { } são obrigatórios.

    (Parece familiar?)

    package main import ( "fmt" "math" ) func sqrt(x float64) string { if x < 0 { return sqrt(-x) + "i" } return fmt.Sprint(math.Sqrt(x)) } func main() { fmt.Println(sqrt(2), sqrt(-4)) }

    If

    Como o for, a instrução if pode começar com uma breve declaração antes de executar a condição.

    Variáveis ​​declaradas pela instrução são válidas somente no escopo até ao final do if.

    (Tente usar v na última instrução return.)

    package main import ( "fmt" "math" ) func pow(x, n, lim float64) float64 { if v := math.Pow(x, n); v < lim { return v } return lim } func main() { fmt.Println( pow(3, 2, 10), pow(3, 3, 20), ) }

    If

    Variáveis ​​declaradas dentro de uma instrução if curta também estão disponíveis dentro dos blocos else.

    package main import ( "fmt" "math" ) func pow(x, n, lim float64) float64 { if v := math.Pow(x, n); v < lim { return v } else { fmt.Printf("%g >= %g\n", v, lim) } // Não pode usar v aqui, pense return lim } func main() { fmt.Println( pow(3, 2, 10), pow(3, 3, 20), ) }

    Structs

    A struct é uma coleção de campos.

    (E uma declaração type faz o que você esperaria.)

    package main import "fmt" type Vertex struct { X int Y int } func main() { fmt.Println(Vertex{1, 2}) }

    Campos Struct

    Campos Struct são acessados ​​através de um ponto.

    package main import "fmt" type Vertex struct { X int Y int } func main() { v := Vertex{1, 2} v.X = 4 fmt.Println(v.X) }

    Ponteiros

    Go tem ponteiros, mas não a aritmética de ponteiros.

    Campos struct podem ser acessados ​​através de um ponteiro struct. A indireção através do ponteiro é transparente.

    package main import "fmt" type Vertex struct { X int Y int } func main() { p := Vertex{1, 2} q := &p q.X = 1e9 fmt.Println(p) }

    Struct literais

    A struct literal denota um valor struct recém-alocado, ao enumerar os valores de seus campos.

    Você pode listar apenas um subconjunto de campos usando o Name: sintaxe. (E a ordem dos campos nomeados é irrelevante.)

    O prefixo especial & constrói um ponteiro para uma struct literal.

    package main import "fmt" type Vertex struct { X, Y int } var ( p = Vertex{1, 2} // tem tipo Vertex q = &Vertex{1, 2} // tem tipo *Vertex r = Vertex{X: 1} // Y:0 é implícito s = Vertex{} // X:0 e Y:0 ) func main() { fmt.Println(p, q, r, s) }

    A função new

    A expressão new(T) aloca um valor zerado para T e retorna um ponteiro para ele. 

    var t *T = new(T)

    or 

    t := new(T)
    package main import "fmt" type Vertex struct { X, Y int } func main() { v := new(Vertex) fmt.Println(v) v.X, v.Y = 11, 9 fmt.Println(v) }

    Maps

    Um map mapeia chaves para valores.

    Maps devem ser criados com make (não com new) antes do uso, o map nil está vazio e não pode ser atribuído.

    package main import "fmt" type Vertex struct { Lat, Long float64 } var m map[string]Vertex func main() { m = make(map[string]Vertex) m["Bell Labs"] = Vertex{ 40.68433, 74.39967, } fmt.Println(m["Bell Labs"]) }

    Maps

    Map literais são como struct literais, mas as chaves são obrigatórias.

    package main import "fmt" type Vertex struct { Lat, Long float64 } var m = map[string]Vertex{ "Bell Labs": Vertex{ 40.68433, -74.39967, }, "Google": Vertex{ 37.42202, -122.08408, }, } func main() { fmt.Println(m) }

    Maps

    Se o tipo de nível superior é apenas um nome do tipo, você pode omiti-lo a partir dos elementos do literal.

    package main import "fmt" type Vertex struct { Lat, Long float64 } var m = map[string]Vertex{ "Bell Labs": {40.68433, -74.39967}, "Google": {37.42202, -122.08408}, } func main() { fmt.Println(m) }

    Mutação de Mapas

    Inserir ou atualizar um elemento no map m: 

    m[key] = elem

    Recuperar um elemento: 

    elem = m[key]

    Excluir um elemento: 

    delete(m, key)

    Teste que uma chave está presente com dois valores: 

    elem, ok = m[key]

    Se key está em m, ok é true. Se não, ok é false e elem tem valor zero pro elemento tipo do map.

    Da mesma forma, durante a leitura de um mapa se a chave não estiver presente, o resultado é o valor zero para o tipo de mapa do elemento.

    package main import "fmt" func main() { m := make(map[string]int) m["Resposta"] = 42 fmt.Println("The value:", m["Resposta"]) m["Resposta"] = 48 fmt.Println("The value:", m["Resposta"]) delete(m, "Resposta") fmt.Println("The value:", m["Resposta"]) v, ok := m["Resposta"] fmt.Println("O valor:", v, "Existe?", ok) }

    Slices

    Um slice aponta para uma matriz de valores e também inclui um comprimento.

    []T é um slice com elementos do tipo T.

    package main import "fmt" func main() { p := []int{2, 3, 5, 7, 11, 13} fmt.Println("p ==", p) for i := 0; i < len(p); i++ { fmt.Printf("p[%d] == %d\n", i, p[i]) } }

    Slices

    Slices podem ser redimencionados, criando um valor de uma nova slice que aponta para a mesma matriz.

    A expressão 

    s[lo:hi]

    avalia uma a uma slice dos elementos de lo através de hi-1, inclusive. Assim 

    s[lo:lo]

    é vazia e 

    s[lo:lo+1]

    tem um elemento.

    package main import "fmt" func main() { p := []int{2, 3, 5, 7, 11, 13} fmt.Println("p ==", p) fmt.Println("p[1:4] ==", p[1:4]) // faltando baixo índice implica 0 fmt.Println("p[:3] ==", p[:3]) // falta implica alto índice len(s) fmt.Println("p[4:] ==", p[4:]) }

    Slices

    Slices são criados com a função make. Ele funciona através da atribuição de uma matriz zerada e retornando uma slice que se refere a essa matriz: 

    a := make([]int, 5)  // len(a)=5
    Slices tem comprimento e capacidade. A capacidade de uma slice é o comprimento máximo da slice pode crescer dentro da matriz subjacente.

    Para especificar uma capacidade, passe um terceiro argumento para make: 

    b := make([]int, 0, 5)
// len(b)=0, cap(b)=5
    Slices podem crescer por "re-slicing" (até sua capacidade): 

    b = b[:cap(b)] // len(b)=5, cap(b)=5
b = b[1:]      // len(b)=4, cap(b)=4
    package main import "fmt" func main() { a := make([]int, 5) printSlice("a", a) b := make([]int, 0, 5) printSlice("b", b) c := b[:2] printSlice("c", c) d := c[2:5] printSlice("d", d) } func printSlice(s string, x []int) { fmt.Printf("%s len=%d cap=%d %v\n", s, len(x), cap(x), x) }

    Slices

    O valor zero de uma fatia é nil.

    Uma fatia nil tem um comprimento e capacidade de 0.

    (Para saber mais sobre as slices, leia o artigo "Slices: internamente e usos" .)

    package main import "fmt" func main() { var z []int fmt.Println(z, len(z), cap(z)) if z == nil { fmt.Println("nil!") } }

    Funções

    Funções são valores também.

    package main import ( "fmt" "math" ) func main() { hypot := func(x, y float64) float64 { return math.Sqrt(x*x + y*y) } fmt.Println(hypot(3, 4)) }

    Funções

    E as funções são closures completos.

    A função somador retorna um closure. Cada closure é obrigado a própria variável soma.

    package main import "fmt" func somador() func(int) int { soma := 0 return func(x int) int { soma += x return soma } } func main() { pos, neg := somador(), somador() for i := 0; i < 10; i++ { fmt.Println( pos(i), neg(-2*i), ) } }

    Range

    O range forma do laço for itera sobre uma slice ou map.

    package main import "fmt" var pow = []int{1, 2, 4, 8, 16, 32, 64, 128} func main() { for i, v := range pow { fmt.Printf("2**%d = %d\n", i, v) } }

    Range

    Você pode ignorar a chave ou valor, atribuindo o _.

    Se você só quer o índice, deixe sem o “, valor ” inteiramente.

    package main import "fmt" func main() { pow := make([]int, 10) for i := range pow { pow[i] = 1<<uint(i) } for _, value := range pow { fmt.Printf("%d\n", value) } }

    Switch

    Você provavelmente sabia que o switch estava chegando.

    O bloco case falhará automaticamente, a menos que termine com uma declaração fallthrough.

    package main import ( "fmt" "runtime" ) func main() { fmt.Print("Go runs on ") switch os := runtime.GOOS; os { case "darwin": fmt.Println("OS X.") case "linux": fmt.Println("Linux.") default: // freebsd, openbsd, // plan9, windows... fmt.Printf("%s.", os) } }

    Switch

    Switch cases avaliam casos de cima para baixo, parando quando um caso for bem-sucedido.

    (Por exemplo, 

    switch i {
case 0:
case f():
}

    não chamar f se i==0.)

    package main import ( "fmt" "time" ) func main() { fmt.Println("Quando é sábado?") today := time.Now().Weekday() switch time.Saturday { case today+0: fmt.Println("Hoje.") case today+1: fmt.Println("Amanhã.") case today+2: fmt.Println("Em dois dias.") default: fmt.Println("Está bem longe.") } }

    Switch

    Switch sem uma condição é o mesmo que switch true.

    Essa construção pode ser uma maneira limpa para escrever longas cadeias if-then-else.

    package main import ( "fmt" "time" ) func main() { t := time.Now() switch { case t.Hour() < 12: fmt.Println("Bom dia!") case t.Hour() < 17: fmt.Println("Boa tarde.") default: fmt.Println("Boa noite.") } }

    Métodos e Interfaces

    Métodos e Interfaces

    Métodos

    Go não tem classes. No entanto, você pode definir métodos em tipos struct.

    O método receptor aparece em sua lista de argumentos entre a própria palavra-chave func e o nome do método.

    package main import ( "fmt" "math" ) type Vertex struct { X, Y float64 } func (v *Vertex) Abs() float64 { return math.Sqrt(v.X*v.X + v.Y*v.Y) } func main() { v := &Vertex{3, 4} fmt.Println(v.Abs()) }

    Métodos

    Na verdade, você pode definir um método em qualquer tipo que você definir em seu pacote, e não apenas estruturas.

    Você não pode definir um método em um tipo de um outro pacote, ou em um tipo básico.

    package main import ( "fmt" "math" ) type MeuFloat float64 func (f MeuFloat) Abs() float64 { if f < 0 { return float64(-f) } return float64(f) } func main() { f := MeuFloat(-math.Sqrt2) fmt.Println(f.Abs()) }

    Métodos com ponteiro receptor

    Os métodos podem ser associados com um tipo nomeado ou um ponteiro para um tipo nomeado.

    Nós só vimos dois métodos Abs. Um no tipo ponteiro *Vertex e outro no valor do tipo MeuFloat.

    Há duas razões para usar um receptor de ponteiro. Primeiro, para evitar copiar o valor de cada chamada de método (mais eficiente se o tipo de valor é uma estrutura grande). Em segundo lugar, de modo que o método possa modificar o valor que o seu receptor aponta.

    Tente mudar as declarações dos métodos Abs e Scale para usar Vertex como o receptor, ao invés de *Vertex.

    O método Scale não tem efeito sobre v é um Vertex. Scale muda v. Quando v é um tipo valor (não-ponteiro), o método vê uma cópia do Vertex e não pode sofrer mutação do valor original.

    Abs funciona de qualquer maneira. Ele apenas lê v. Não importa se ele está lendo o valor original (através de um ponteiro) ou uma cópia desse valor.

    package main import ( "fmt" "math" ) type Vertex struct { X, Y float64 } func (v *Vertex) Scale(f float64) { v.X = v.X * f v.Y = v.Y * f } func (v *Vertex) Abs() float64 { return math.Sqrt(v.X*v.X + v.Y*v.Y) } func main() { v := &Vertex{3, 4} v.Scale(5) fmt.Println(v, v.Abs()) }

    Interfaces

    Um tipo de interface é definida por um conjunto de métodos.

    Um valor de tipo de interface pode conter qualquer valor que implementa esses métodos.

    package main import ( "fmt" "math" ) type Abser interface { Abs() float64 } func main() { var a Abser f := MeuFloat(-math.Sqrt2) v := Vertex{3, 4} a = f // o MeuFloat implementa Abser a = &v // a *Vertex implementa Abser a = v // um Vertex, NÃO // implementa Abser fmt.Println(a.Abs()) } type MeuFloat float64 func (f MeuFloat) Abs() float64 { if f < 0 { return float64(-f) } return float64(f) } type Vertex struct { X, Y float64 } func (v *Vertex) Abs() float64 { return math.Sqrt(v.X*v.X + v.Y*v.Y) }

    Interfaces

    Um tipo implementa uma interface através da implementação dos métodos.

    Não há declaração explícita de intenções.

    As interfaces implícitas dissociam os pacotes de implementação que definem as interfaces: um não depende do outro.

    Além disso, incentiva a definição de interfaces precisas, porque você não tem que encontrar todas as implementações e marcá-las com o novo nome de interface.

    Pacote io define Reader e Writer; você não precisa implementar.

    package main import ( "fmt" "os" ) type Reader interface { Read(b []byte) (n int, err error) } type Writer interface { Write(b []byte) (n int, err error) } type LeiaEscritor interface { Reader Writer } func main() { var w Writer // os.Stdout implementa Escritor w = os.Stdout fmt.Fprintf(w, "olá, escritor\n") }

    Erros

    Goroutines

    Uma goroutine é um segmento leve e gerenciado pelo runtime de Go. 

    go f(x, y, z)

    inicia uma nova execução goroutine 

    f(x, y, z)

    A avaliação de f, x, y, and z acontece no goroutine corrente e para a execução de f acontece no goroutine novo.

    Goroutines executam no mesmo espaço de endereço, para acesso à memória compartilhada seja sincronizada. O pacote de sincronia fornece as primitivas úteis, embora você não vá precisar muito deles em Go, pois há outras primitivas. (Veja o próximo slide).

    package main import ( "fmt" "runtimetime" ) func say(s string) { for i := 0; i < 5; i++ { runtime.Gosched()time.Sleep(100 * time.Millisecond) fmt.Println(s) } } func main() { go say("mundo") say("olá") }

    Canais

    Canais são um canal digitado através do qual você pode enviar e receber valores com o operador de canal, <-. 

    ch <- v    // v envia para o canal ch.
v := <-ch  // Recebe do pc, e
           // atribui o valor de v

    (Os dados fluem na direção da seta.)

    Como maps e slices, os canais devem ser criados antes de se usar: 

    ch := make(chan int)

    Por padrão, envia e recebe bloco até o outro lado estar pronto. Isso permite que goroutines sincronizem sem bloqueios explícitos ou variáveis ​​de condição.

    package main import "fmt" func soma(a []int, c chan int) { soma := 0 for _, v := range a { soma += v } c <- soma // envia soma para c } func main() { a := []int{7, 2, 8, -9, 4, 0} c := make(chan int) go soma(a[:len(a)/2], c) go soma(a[len(a)/2:], c) x, y := <-c, <-c // recebe de c fmt.Println(x, y, x + y) }

    Canais Bufferizados

    Os canais podem ser bufferizados. Fornecendo o tamanho do buffer como o segundo argumento para make para inicializar um canal bufferizado: 

    ch := make(chan int, 100)

    Envia para um bloco de canais bufferizados apenas quando o buffer estever cheio. Recebe bloco quando o buffer está vazio.

    Modifique o exemplo, enchendo o buffer e veja o que acontece.

    package main import "fmt" func main() { c := make(chan int, 2) c <- 1 c <- 2 fmt.Println(<-c) fmt.Println(<-c) }

    Range e Close

    Um remetente pode close um canal para indicar que os valores não serão mais enviados. Receptores pode testar se um canal foi fechado através da atribuição de um segundo parâmetro para a expressão de recepção: depois 

    v, ok := <-ch

    ok é false se houver valores não mais receber e que o canal está fechado.

    O laço for i := range c recebe valores do canal repetidamente até que seja fechado.

    Nota: Apenas o remetente deve fechar um canal, nunca o receptor. O envio em um canal fechado irá causar um erro pânico.

    Outra nota: Canais não são como arquivos, você geralmente não precisa fechá-los. O encerramento só é necessário quando o receptor precisa saber que não há mais valores chegando, como para terminar um laço range.

    package main import ( "fmt" ) func fibonacci(n int, c chan int) { x, y := 1, 1 for i := 0; i < n; i++ { c <- x x, y = y, x + y } close(c) } func main() { c := make(chan int, 10) go fibonacci(cap(c), c) for i := range c { fmt.Println(i) } }

    Select

    A instrução select permite uma espera na goroutine sobre as operações de comunicação múltiplas.

    O bloco select aguarda até que um de seus cases possam executar, então ele executa esse case. Ele escolhe um ao acaso se vários estiverem prontos.

    package main import "fmt" func fibonacci(c, quit chan int) { x, y := 1, 1 for { select { case c <- x: x, y = y, x + y case <-quit: fmt.Println("sair") return } } } func main() { c := make(chan int) quit := make(chan int) go func() { for i := 0; i < 10; i++ { fmt.Println(<-c) } quit <- 0 }() fibonacci(c, quit) }

    Selection Padrão

    O case default em um select é executado se nenhum outro caso está pronto.

    Utilize um case default para tentar um enviar ou receber sem bloqueio: 

    select {
case i := <-c:
	// use i
default:
	// Recebendo c bloquearia
}

    Nota: Este exemplo não será executado através da interface de usuário baseada no passeio web porque o ambiente sandbox não tem noção de tempo. Você pode querer instalar Go para ver este exemplo em ação.

    package main import ( "fmt" "time" ) func main() { tick := time.Tick(1e8) boom := time.After(5e8) for { select { case <-tick: fmt.Println("tick.") case <-boom: fmt.Println("BOOM!") return default: fmt.Println(" .") time.Sleep(5e7) } } }
